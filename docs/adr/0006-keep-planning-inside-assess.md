# Keep Planning Inside Assess

`/ultra solve` should keep `Ticket Discovery` as a lightweight pre-claim eligibility scan, then claim selected tickets before deeper exploration. The Pre-Implementation Checkpoint belongs inside `Assess` after claim: the agent re-reads the ticket, acceptance criteria, comments, linked docs, Agent Brief, and code context; performs adaptive read-only exploration; classifies missing information or human-required decisions; and writes an Execution Digest only when it changes future execution, delegation, recovery, or review.
