# Use Execution Digests for Complex Solves

Substantial `/ultra solve` planning should persist a compressed Execution Digest outside the Ticket at `.scratch/<feature>/execution-digests/<digest-key>.md` instead of raw planning notes or subagent transcripts. The digest is written after claim and before implementation edits, so delegated, decisionful, risk-bearing, or resumable solve runs stay executable by preserving strategy, touched surfaces, risks, validation, and record-worthy decisions or deviations, while avoiding context pollution and tracker noise from full exploration logs. Execution Digests remain ticket-level; grouped execution does not create grouped digests by default because ready-for-agent tickets should already be small enough to resume and review independently. Durable digest items are distilled into the outcome Solve Record, and a digest without resume value is removed.
