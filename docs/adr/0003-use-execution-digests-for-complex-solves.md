# Use Execution Digests for Complex Solves

Substantial `/ultra solve` planning should persist a compressed Execution Digest on the ticket instead of raw planning notes or subagent transcripts. The digest is written after claim and before implementation edits, so delegated, decisionful, risk-bearing, or resumable solve runs stay executable by preserving strategy, touched surfaces, risks, validation, and agent decisions, while avoiding context pollution and tracker noise from full exploration logs. Execution Digests remain ticket-level; grouped execution does not create grouped digests by default because ready-for-agent tickets should already be small enough to resume and review independently. Finished results, checks, manual gates, and remaining caveats belong in the solve record rather than being copied from the ticket digest.
